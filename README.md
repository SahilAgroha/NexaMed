## рҹ§  What is NexaMed?

**NexaMed** is a **production-grade, AI-powered adaptive learning platform** built for medical students and healthcare professionals. It's not just another CRUD app вҖ” it's a full microservices ecosystem that learns from each student, conducts AI-driven interviews, and facilitates live video mentoring sessions.

### The Problem
> Traditional medical education platforms are static. They don't adapt to individual student weaknesses, don't provide real interview preparation, and don't give actionable feedback. NexaMed fixes all of this.

### The Solution
```
Student learns вҶ’ AI tracks gaps вҶ’ Platform adapts content вҶ’ 
AI interviews student вҶ’ Live mentor reviews вҶ’ Detailed reports generated вҶ’ 
Student improves вҶ’ Repeat
```

## вңЁ Key Features

### рҹҺ“ Adaptive Learning Engine
- **AI-Generated Quizzes** вҖ” GPT-4 creates unique questions per topic and difficulty
- **Adaptive Difficulty** вҖ” Platform auto-adjusts based on real-time performance data
- **Clinical Case Simulator** вҖ” Step-by-step patient scenarios with AI diagnosis feedback
- **Peer Q&A System** вҖ” Stack Overflow-style discussions within courses
- **Real-time Study Rooms** вҖ” Collaborative sessions via WebSockets
- **Progress Heatmaps** вҖ” Visual tracking of strong and weak topic areas

### рҹҺҘ Dual Interview System *(Unique Feature)*

#### рҹӨ– AI Mock Interview (Solo Mode)
```
Student starts session
      вҶ“
GPT-4 generates medical questions based on course progress
      вҶ“
Student answers verbally вҶ’ Whisper API transcribes in real-time
      вҶ“
AI evaluates: clarity + accuracy + hesitation + response time
      вҶ“
Detailed PDF report with improvement tips generated
      вҶ“
Session saved to AWS S3 for self-review
```

#### рҹ‘Ҙ Live Interview (Human Mode)
```
Student books slot via Google Calendar
      вҶ“
WebRTC peer-to-peer video call initiated
      вҶ“
Interviewer gets live scoring sheet + AI-suggested follow-ups
      вҶ“
Post-call: AI combines human scores вҶ’ final report
```

- Cohort-level performance insights
- Exportable PDF reports

### рҹ”” Real-time Notifications
- Interview reminders via Google Calendar
- Course updates pushed via Kafka вҶ’ WebSocket

## рҹҡҖ Getting Started

### Prerequisites

Make sure you have these installed:

```bash
java --version        # Java 17+
mvn --version         # Maven 3.8+
node --version        # Node.js 18+
docker --version      # Docker 24+
docker compose version # Docker Compose v2
```

### 1. Clone the Repository

```bash
git clone https://github.com/yourusername/nexamed.git
cd nexamed
```

### 2. Configure Environment Variables

```bash
cp .env.example .env
```

Open `.env` and fill in your keys:

```env
# JWT
JWT_SECRET=your-super-secret-jwt-key-min-256-bits

# Google OAuth2
GOOGLE_CLIENT_ID=your-google-client-id.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=your-google-client-secret

# OpenAI
OPENAI_API_KEY=sk-your-openai-api-key

# AWS S3
AWS_ACCESS_KEY_ID=your-aws-access-key
AWS_SECRET_ACCESS_KEY=your-aws-secret-key
AWS_S3_BUCKET=nexamed-assets
AWS_REGION=us-east-1
```

### 3. Start Infrastructure

```bash
# Start PostgreSQL, Redis, Kafka in background
docker compose up postgres redis zookeeper kafka -d

# Verify all containers are healthy
docker compose ps
```

### 4. Run Backend Services

```bash
# Option A: Run all at once with Docker
docker compose up --build

# Option B: Run individually for development
cd backend/auth-service    && mvn spring-boot:run &
cd backend/course-service  && mvn spring-boot:run &
cd backend/ai-service      && mvn spring-boot:run &
cd backend/interview-service && mvn spring-boot:run &
```

### 5. Run Frontend

```bash
cd frontend
npm install
npm run dev
```

### 6. Open the App

```
рҹҢҗ Frontend:     http://localhost:3000
рҹ”Ң API Gateway:  http://localhost:8080
рҹ“Ҡ Kafka UI:     http://localhost:9021   (if enabled)
```

> Full Swagger UI available at: `http://localhost:8080/swagger-ui.html`

### Core Endpoints

```
Auth Service
  POST   /api/auth/register          Register new user
  POST   /api/auth/login             Login & get JWT token
  GET    /api/auth/oauth2/google     Google OAuth2 login

Course Service
  GET    /api/courses                List all published courses
  POST   /api/courses                Create course (Teacher only)
  POST   /api/courses/{id}/enroll    Enroll in a course
  GET    /api/courses/{id}/progress  Get student progress

AI Service
  POST   /api/ai/quiz/generate       Generate quiz for a topic
  POST   /api/ai/cases/simulate      Start clinical case session
  POST   /api/ai/evaluate            Evaluate interview transcript

Interview Service
  POST   /api/interviews/mock/start  Start AI mock interview
  GET    /api/interviews/{id}/report Get interview report
  POST   /api/interviews/live/book   Book a live interview slot
  WS     /ws/interview/{roomId}      WebRTC signaling channel
```

## рҹҢҚ Deployment

### Deploy with Docker Compose (Production)

```bash
# Build all images
docker compose -f docker-compose.prod.yml build

# Deploy
docker compose -f docker-compose.prod.yml up -d

# View logs
docker compose logs -f api-gateway
```

### CI/CD Pipeline (GitHub Actions)

Every push to `main` automatically:
1. вң… Runs all unit & integration tests
2. рҹҸ—пёҸ Builds all Spring Boot JARs
3. рҹҗі Builds Docker images
4. рҹҡҖ Deploys to production server via SSH
